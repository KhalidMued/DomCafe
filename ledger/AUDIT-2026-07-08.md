# Production-Readiness Audit — 2026-07-08

Read-only audit of frontend, backend, security, performance, Docker deployment, API design, database, UI/UX, accessibility, and mobile responsiveness. Findings verified against source and the live stack where possible. No rewrites recommended; all fixes preserve the existing architecture.

## Summary

The architecture is clean and consistently layered, input validation is thorough, auth boundaries are respected, and soft-deletes plus order snapshots are correct. The gaps cluster in operational resilience, two real security issues, a handful of guest-facing UX bugs, and moderate technical debt.

## Findings

### Critical

- **C1 — Stack does not self-heal.** No `restart:` policies, no `healthcheck:` blocks, no `depends_on` readiness conditions in `docker-compose.yml`. Already caused the recorded Cloudflare 502 outage.
- **C2 — No regression safety net.** No CI workflows existed; the backend test suite was not runnable on the host (no pytest environment) nor in the image (tests not copied). The prior "67 passed" baseline was not reproducible.

### High

- **H1 — Order enumeration.** `GET /api/orders/{order_id}` is unauthenticated with sequential integer IDs; anyone can walk all orders and read guest names/notes. Fix: random `public_code` lookup column.
- **H2 — Rate limits share one global bucket.** Backend `_client_ip()` sees only the Nginx container IP (uvicorn lacks `--proxy-headers`), so "per-IP" limits are site-wide: 10 orders/min total. Nginx `$remote_addr` behind cloudflared likely has the same issue. Fix: proxy headers + real-IP configuration.
- **H3 — Security headers dropped on `/api/*` and `/uploads/*` (verified live).** Nginx `add_header` inheritance: any location with its own `add_header` loses the server-level security headers. Uploaded images are served without `nosniff`.
- **H4 — One failed poll erases the guest's active order.** `MenuPage.tsx` clears `activeOrderId` on any fetch error, not just 404.

### Medium

- M1 — Discord webhook awaited inline in order creation (up to +5s per order); move to a background task.
- M2 — `api.ts request()` calls `response.json()` unconditionally; nginx 429/502 HTML responses crash error handling.
- M3 — Cart is memory-only; page refresh loses it (mobile QR flow). Persist to sessionStorage.
- M4 — Cart quantity stepper has no max; backend caps at 10 → generic 422.
- M5 — Google Fonts CSS `@import` is render-blocking and external; self-host fonts.
- M6 — No backend logging/request-ID middleware exists (ledger previously claimed it did); only uvicorn defaults.
- M7 — `get_redis()` opens a new connection per call; use a shared client.
- M8 — Whitespace-only guest names pass validation (strip after validate).
- M9 — A drink's `default_bean_id` cannot be cleared via PATCH.
- M10 — Admin JWT in localStorage with 24h expiry (acceptable trade-off; note httpOnly cookie as stronger).
- M11 — Admin nav links bypass the SPA router (full page reloads); logout uses a pushState hack.
- M12 — No fetch timeouts/retry/backoff/offline handling (promised in AGENT.md §26, never implemented); status page retries 404s forever.
- M13 — Backend container ran as root and copied all of `docs/` into the image.
- M14 — No order-status timestamps or transition rules (spec's `received_at/preparing_at/ready_at` dropped).

### Low

- L1 — Dead code: `app/router.tsx`, `app/providers.tsx`, `lib/{errors,validators,i18n}.ts`, seven unprefixed admin page placeholders, `_current_admin_dependency` TypeError wrapper, `getattr(redis, "eval")` shim, unused `passlib` dependency, unused `VITE_API_BASE_URL` compose env, redundant `docker-compose.prod.yml`.
- L2 — Duplicate logic: `_as_bool` ×3; payload builders duplicated between `admin_menu.py` and `agent.py`; status labels duplicated frontend/backend.
- L3 — Replaced drink photos never deleted from `uploads/drinks/`.
- L4 — Admin login timing oracle (no dummy hash for unknown users).
- L5 — PgBouncer `AUTH_TYPE: plain` (internal-only; scram preferred).
- L6 — UX polish: no cancelled styling on OrderStatusPage; collidable list keys; drink photos `alt=""`; `#888780` text ≈ 3.9:1 contrast (below AA small-text).
- L7 — Perf: API JSON not gzipped at edge; no image resize/WebP pipeline; 5 sequential dashboard COUNTs.
- L8 — `config.py` `env_file="../.env"` resolves nowhere in-container (compose env injection masks it).
